# Events Sequence (2026-01-28)

아주 쉬운 말: 버튼을 누르면 어떤 일이 차례대로 일어나는지 순서예요.

## 학습 만들기 (Main)
1) 사용자가 메인에서 `학습 만들기` 버튼 클릭 → 모달 `open=true`
2) 제목 입력 후 제출(submit)
3) `addLearning(title)` 호출 → 상태에 새 항목 추가
4) 모달 닫힘 → 최근 목록이 자동으로 업데이트되어 새 항목 보임

## 학습 삭제/참여 토글/진척도 변경 (Main/My)
- 삭제: `removeLearning(id)` → 목록에서 제거
- 참여 토글: `toggleJoined(id)` → `joined` true/false 반전
- 진척도 변경: `updateProgress(id, v)` → 0~100로 클램프하여 반영

## 헤더 버튼 전환
- 현재 경로 확인(`/` vs `/mypage`) → 라벨/링크를 `MY`↔`MAIN`으로 설정

## 5살 설명
- 만들기 버튼을 누르면 새 카드가 짠! 하고 생겨요. 필요 없으면 지우고, 하고 있으면 체크하고, 얼마나 했는지 숫자를 움직여요. 위의 버튼으로 메인과 마이 사이를 쉽게 왔다갔다 해요.
