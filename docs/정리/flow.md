# 우리 앱이 어떻게 움직일까? (아기도 알 수 있게)

- 이 문서는 아주 쉬운 말로, 화면이 어떻게 만들어지는지 순서를 알려줘요.
- 읽으면 "아, 이렇게 해서 화면이 보이는구나!" 하고 알 수 있어요.

## 순서
- 1) 문을 열어요: `app/layout.tsx`
  - 화면의 집을 만들어요. 위에 머리(헤더), 가운데(내용), 아래(푸터)가 있어요.
  - 집 안에는 불빛을 밝게/어둡게 바꾸는 도우미(Providers)가 있어요.
- 2) 방을 들어가요: `app/page.tsx`
  - 오늘 해야 할 일, 수업 시간표, 진행률 같은 내용을 보여줘요.
  - 길~게 적힌 글은 너무 길면 "..."으로 줄여서 예쁘게 보여줘요.
- 3) 불빛 스위치: `components/theme-toggle.tsx`
  - 버튼을 누르면 밝게/어둡게 바뀌어요. (낮 모드 / 밤 모드)
- 4) 스위치를 움직이게 해주는 상자: `components/providers.tsx`
  - 화면의 색(테마)을 바꾸는 도우미예요.

## 화면이 보이는 흐름
- 브라우저가 켜져요 → `layout`이 집을 만들어요 → `page`가 내용을 넣어요 →
  필요하면 스위치(테마 버튼)를 눌러 색을 바꿔요 → 길이가 긴 글은 "..."으로 예쁘게 줄어요.

## 중요한 약속
- `layout`은 집(틀)만 만들어요. 여기서는 스위치를 직접 만들지 않아요.
- 스위치는 `Providers` 안에서만 써요. 그래야 잘 동작해요.
- 너무 긴 글은 `truncate`(말줄임)로 "..."이 보이게 만들어요.

## 단어 사전
- 레이아웃(layout): 화면의 큰 틀, 집 같은 것.
- 페이지(page): 집 안의 방, 내용을 보여주는 곳.
- 테마(theme): 밝게/어둡게 같은 화면 색 느낌.
- 말줄임표(ellipsis): 글이 너무 길면 "..."으로 보여주는 것.
